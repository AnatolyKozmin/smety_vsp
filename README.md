# Вспышка'26 — сметы и закупки

Веб-приложение для планирования сметы выезда: продукты → блюда → меню по дням → список покупок, смета и взносы.

- **🥕 Продукты** — каталог: название, ед. изм., граммы в упаковке, цена, срок хранения
- **📖 База блюд** — рецепты со ссылками на продукты + граммовка на 1 порцию
- **🧑‍🤝‍🧑 Участники** — люди с ролями
- **🏕️ Забросы** — для каждого выезда: дни → приёмы пищи → блюда → участники + «Прочее»
- **🍲 Меню** — сборка меню в одном месте
- **🛒 Что купить** — сводный список покупок: упаковок и сумма, разделено на долгосрочное/краткосрочное
- **📊 Смета** и **💰 Взносы** — автоматический расчёт стоимости и долей по людям

Стек: Vue 3 + Vite, FastAPI + SQLAlchemy, SQLite.

## Локальная разработка

### Бэкенд

```bash
cd backend
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/python seed.py          # один раз — наполнить БД из xlsx (если нужно)
.venv/bin/uvicorn app.main:app --reload --port 8765
```

API: `http://127.0.0.1:8765`. Swagger: `/docs`.

### Фронтенд

```bash
cd frontend
npm install
npm run dev    # http://localhost:3000 (порт зашит в vite.config.js)
```

Vite проксирует `/api` на `127.0.0.1:8765`.

## Деплой в Docker на удалённый сервер

Два контейнера: `backend` (Python+uvicorn) и `frontend` (nginx со статикой + проксированием `/api`).
SQLite-файл живёт на хосте в `./data/smety.db` — переживает пересборки.

### Что нужно на сервере
- Docker и Docker Compose (`docker compose version`)
- Свободный порт (по умолчанию `8082`, переопределяется через `HTTP_PORT`)

### Первый деплой

```bash
# 1) На сервере: получаем код
git clone <your-repo-url> /srv/smety
cd /srv/smety

# 2) Готовим директорию для БД
mkdir -p data
```

**Перенос локальной БД на сервер** (с локальной машины):

```bash
# с локальной машины:
scp backend/smety.db user@server:/srv/smety/data/smety.db
```

Если переноса не требуется — пустую БД создаст бэкенд при старте; можно затем выполнить
`docker compose exec backend python seed.py`, чтобы залить блюда/продукты/людей из xlsx.

```bash
# 3) Сборка и запуск
docker compose up -d --build

# Логи
docker compose logs -f
```

Открываем `http://<server>:8082/`.

### Обновление кода

```bash
git pull
docker compose up -d --build
```

БД на хосте (`./data/smety.db`) не трогается.

### Поменять порт

```bash
HTTP_PORT=8090 docker compose up -d --build
# или сохранить в .env:
echo "HTTP_PORT=8090" > .env
```

### Бэкап БД

```bash
# на сервере
cp data/smety.db data/smety.db.$(date +%F).bak
# или скачать к себе
scp user@server:/srv/smety/data/smety.db ./backup-$(date +%F).db
```

### HTTPS

Поднять reverse-proxy (Caddy / nginx-proxy / Traefik) перед контейнером `frontend` или вынести его за reverse-proxy на хосте. Скорее всего проще всего:

```yaml
# в docker-compose.yml оставить
ports:
  - "127.0.0.1:8080:80"
```

И на хосте Caddy/nginx с автоматическим Let's Encrypt, проксирующий на `127.0.0.1:8080`.

## Файлы данных и git

`.gitignore` исключает любые `*.db` и директорию `data/` — то есть:
- **Локальная БД** разработки: `backend/smety.db` — не попадает в git
- **Продовая БД**: `./data/smety.db` (volume на сервере) — тоже не в git
- Они **независимы** друг от друга

Перенос продовой БД между машинами — через `scp` или ручную копию файла.

## Логика расчётов

- **Всего, г** = `г_на_порцию × количество_участников_приёма_пищи`
- **Сколько надо взять** = `roundup(всего_г / г_в_упаковке)`
- **Итог цена ингредиента** = `сколько_взять × цена_за_упаковку`
- **Итог дня/приёма** = сумма ингредиентов
- **С наценкой** = `база × (1 + наценка_% / 100)`
- **Доля на человека** для приёма пищи = `итог_приёма / число_участников`
- **Взнос участника** = сумма его долей по приёмам, где он отмечен, + доля от «Прочего» (если отмечен в нём), всё умножить на (1 + наценка/100)
- **Статус оплаты**: `Оплачено` если `paid ≥ amount`, `Частично` если `0 < paid < amount`, иначе `—`
- **Список покупок** (ИТОГО): для каждого продукта суммируется расход со всех дней/блюд + «Прочее» → округляется вверх до упаковок → умножается на цену; разделено на «Краткосрочное» (мясо/овощи) и «Долгосрочное» (крупы/консервы).
